# Supplier Credit Risk & Cash Flow Analysis | Python

# Project Overview
This project analyzes supplier transactions to evaluate payment behavior, outstanding liabilities, and potential credit risk. Using Python, the project processes financial ledger data, identifies payment trends, and generates insights that can help organizations improve cash flow management and make informed financial decisions.

# Objective
The objective of this project is to analyze supplier payment data, identify suppliers with higher credit risk, monitor outstanding balances, and forecast future liabilities using historical transaction data.

# Tools & Technologies
* Python
* Pandas
* NumPy
* Matplotlib

# Dataset
The project uses a financial ledger dataset containing **234 real-world supplier transactions** with information such as:
* Supplier Name
* Transaction Date
* Purchase Amount
* Payment Amount
* Outstanding Balance
* Payment Status

# Features
* Data cleaning and preprocessing
* Transaction summary and financial analysis
* Supplier-wise purchase analysis
* Payment behavior analysis
* Outstanding liability tracking
* Credit risk identification
* Data visualization using charts and graphs
* Cash flow trend analysis
* Future outstanding balance prediction using Linear Regression

# Key Analysis Performed
* Calculated total purchases and payments.
* Identified suppliers with high outstanding balances.
* Analyzed payment delays and supplier payment behavior.
* Evaluated supplier credit risk based on outstanding liabilities.
* Forecasted future outstanding balances using Linear Regression.

# Key Insights
* Identified suppliers with consistently high outstanding balances.
* Detected payment patterns that could indicate potential credit risk.
* Visualized purchase and payment trends over time.
* Generated forecasts to support future cash flow planning.
